Scénario 06

Soumission Vendeur

Le vendeur n'est pas forcément à l'aise avec le numérique. Il peut être un commerçant, une ménagère, un étudiant. Le formulaire doit être le plus simple possible — 3 étapes, des mots simples, pas de jargon. Il ne publie rien lui-même. Il soumet une demande et attend qu'on vienne chez lui. C'est tout.

Écran 1 Accueil Vendeur

Explication visuelle du processus en 3 étapes simples avant même de commencer. Le vendeur sait exactement ce qui va se passer avant de toucher quoi que ce soit.

→
3 étapes illustrées : Vous décrivez → On vient chez vous → Vous recevez votre argent. Clair, rassurant, humain.
✓
Si le vendeur a déjà des demandes en cours, elles apparaissent en bas avec leur état actuel.
!
Si non connecté → tap sur "Soumettre" → modal connexion. Après connexion → retour direct ici, rien perdu.
→
Lien discret "Comment ça marche ?" pour ceux qui veulent plus d'explications avant de commencer.

Écran 2 Formulaire — Étape 1/3 : Le produit

Catégorie, nom, état, quantité, description optionnelle. Les boutons d'état sont visuels — pas une liste déroulante. L'utilisateur comprend d'un coup d'œil.

✓
Bouton "Suivant" inactif tant que catégorie, nom et état ne sont pas remplis. Il s'active progressivement.
→
Sélection de catégorie → change l'icône du produit dans le formulaire pour confirmer visuellement le choix.
!
Quantité max = 10 par soumission. Au-delà, message : "Pour une grande quantité, contactez-nous directement."
→
Description libre optionnelle. Le placeholder donne des exemples concrets adaptés au contexte camerounais.

Écran 3 Formulaire — Étape 2/3 : Le prix

Le vendeur saisit son prix. La commission et le montant qu'il recevra se calculent en temps réel. Transparence totale. Aucune surprise.

✓
Calcul en temps réel : Prix vendeur → Commission → Montant reçu. Se met à jour à chaque chiffre tapé.
→
Photos optionnelles (5 max). Message clair : "Les photos officielles seront prises par notre agent. Les vôtres servent d'aperçu."
!
Prix trop bas pour la catégorie → alerte douce : "Ce prix semble bas pour ce type de produit. Êtes-vous sûr ?"
✗
Prix à 0 ou vide → bouton Suivant reste inactif. Jamais de soumission sans prix.

Écran 4 Formulaire — Étape 3/3 : Adresse & Disponibilité

Zone de couverture, adresse descriptive (pas de code postal — contexte camerounais), créneaux de disponibilité, numéro de contact pré-rempli.

✗
Zone non couverte → message immédiat : "On n'est pas encore dans votre zone. Laissez votre numéro, on vous prévient dès qu'on arrive." Jamais bloquer sans alternative.
✓
Adresse libre en texte — placeholder contextuel : "Ex: Rue des palmiers, en face de l'école, maison bleue". Réalité camerounaise respectée.
→
Numéro de contact pré-rempli avec le numéro du compte. Modifiable si besoin (ex: numéro d'un proche à contacter).
!
Au moins un créneau de disponibilité obligatoire. Sinon le bouton reste inactif.

Écrans 5 & 6 Confirmation & Suivi Vendeur

Confirmation rassurante avec récapitulatif financier clair. Timeline vendeur avec 6 états. Lien vers l'annonce en lecture seule quand le produit est mis en vente.

✓
Récapitulatif confirme : prix demandé, commission, montant estimé reçu. Aucune surprise ultérieure.
→
Notification push à chaque étape. Ex: "Votre agent arrive demain matin entre 8h et 12h. Soyez disponible."
!
Produit en vente → lien "Voir mon annonce" affiché. Vendeur voit sa fiche en lecture seule. Ne peut rien modifier.
✗
Annulation possible uniquement avant collecte. Après que l'agent a récupéré le produit → annulation impossible via app. Contact support requis.
Scénario 07

Application Agent Terrain

L'agent terrain est le cœur opérationnel de la plateforme. Sans lui, rien ne fonctionne. Son application doit être ultra-simple, rapide, utilisable d'une main sur son téléphone, même en déplacement. Chaque mission est documentée, tracée, horodatée. Zéro ambiguïté sur ce qu'il doit faire à chaque étape.

Écran 1 Tableau de bord Agent

Vue d'ensemble immédiate : missions du jour, missions urgentes en premier, statistiques de performance. L'agent sait en un coup d'œil ce qu'il doit faire.

→
Missions triées par urgence : livraisons du jour en haut, collectes planifiées ensuite.
✓
Badge rouge sur l'icône missions si nouvelle mission assignée pendant que l'agent est connecté.
!
Mission expirée (délai dépassé) → alerte rouge. Admin notifié automatiquement.
→
Statistiques du mois : missions complétées, taux de réussite, zones couvertes.

Écrans 2 & 3 Mission Collecte — Vérification

L'agent suit une checklist obligatoire. Chaque point doit être validé. Photos prises in-app uniquement — pas depuis la galerie. Tout est tracé, horodaté, géolocalisé.

→
Bouton GPS intégré → ouvre Maps avec l'adresse du vendeur pré-remplie. Un seul tap.
✓
Checklist de vérification : CNI vendeur ✓, Preuve achat ✓, Correspondance produit ✓. Toutes obligatoires.
✗
Photos depuis galerie interdites. Uniquement prises in-app avec horodatage et géolocalisation automatique. Impossibilité de tricher.
!
Si propriété non prouvée → bouton "Refuser la collecte". Raison obligatoire. Admin notifié immédiatement. Vendeur notifié avec explication.
→
Minimum 4 photos obligatoires : face, dos, côtés, défauts visibles. L'app refuse de continuer sans.

Écran 4 Mission Livraison

L'agent livre le produit à l'acheteur. Confirmation de livraison avec code unique que l'acheteur doit présenter. Signature ou code — la remise est prouvée.

→
L'acheteur reçoit un code de confirmation à 6 chiffres. L'agent saisit ce code pour confirmer la remise. Les deux doivent correspondre.
✓
Code validé → livraison confirmée. Statut mis à jour instantanément. Acheteur et admin notifiés.
✗
Acheteur absent après 2 tentatives → agent signale "Acheteur injoignable". Admin prend le relais pour replanifier.
!
Produit endommagé constaté lors de la livraison → agent signale avant remise. Photo obligatoire. Remise suspendue jusqu'à décision admin.
Scénario 08

Panel Admin

L'admin est le chef d'orchestre quotidien. Il valide, publie, assigne, gère les litiges, suit les commandes. Son interface est un tableau de bord web dense mais organisé. Chaque information est accessible en maximum 2 clics. Aucune action destructive sans confirmation explicite.

Écran 1 Tableau de bord Admin — Vue générale

KPIs du jour en haut. Alertes prioritaires en rouge. Demandes en attente, commandes en cours, missions agents — tout visible sans naviguer.

✗
Alerte rouge si demande vendeur en attente depuis plus de 4h. L'admin doit traiter en priorité.
!
Alerte orange si livraison en retard de plus de 2h. Action requise immédiatement.
✓
Toutes les métriques clés visibles d'un coup d'œil : revenus du jour, commandes actives, produits en vente, agents en mission.
→
Chaque chiffre cliquable → vue détaillée. Ex: tap sur "3 réclamations" → liste des réclamations ouvertes.

Écran 2 Publication d'un produit

L'admin rédige la fiche produit complète après collecte. Toutes les sections obligatoires. Les défauts doivent être renseignés — jamais laisser vide sans raison. La fiche ne peut être publiée qu'avec photos officielles de l'agent.

✗
Publication impossible sans photos officielles de l'agent. Les photos du vendeur ne suffisent pas.
✓
Bloc défauts obligatoire. 3 options : liste des défauts / "Aucun défaut visible" / "Défauts internes non garantis". L'une des trois doit être cochée.
→
Prix affiché = Prix vendeur. La commission est calculée automatiquement selon la catégorie. L'admin ne fixe pas le prix.
!
Prévisualisation de la fiche avant publication. L'admin voit exactement ce que verra l'acheteur.
→
Après publication → notification automatique au vendeur : "Votre produit est maintenant en vente."
⚙️

Autres modules Admin : Gestion des commandes (filtres par état, assignation livraison), Gestion des utilisateurs (acheteurs et vendeurs, suspension de compte), Gestion des agents (zones, missions, performances), Litiges et réclamations (décision remboursement, clôture), Paiements et reversements (historique, export comptable).

Scénario 09

Panel Super Admin

Le Super Admin est le seul à avoir une vue à 360° sur toute l'entreprise. Il configure les règles, gère les admins, consulte les rapports financiers stratégiques. Il n'opère pas au quotidien — il pilote, analyse, décide. Son interface est orientée données et configuration, pas opérations.

Écran 1 Tableau de bord Super Admin — Vue stratégique

→
Seul à voir les revenus nets réels après reversements vendeurs. L'admin ordinaire ne voit que les commandes.
✓
Logs complets de toutes les actions admin. Qui a fait quoi, quand, depuis quelle IP.
✗
Suppression d'un compte admin → confirmation en deux étapes + mot de passe Super Admin requis. Aucune action irréversible en un seul clic.
!
Configuration des commissions → effet immédiat sur tous les nouveaux produits. Produits déjà publiés conservent leur ancienne commission.
→
Export des données en CSV/Excel pour analyses externes. Rapports hebdomadaires auto-générés et envoyés par email.

⚙️ Configuration système

  • Taux de commission par catégorie
  • Zones géographiques couvertes
  • Délais d'expiration réservation
  • Prix livraison par zone
  • Délai de réclamation post-livraison
  • Messages automatiques personnalisables

💰 Rapports financiers

  • Revenus nets par jour/semaine/mois
  • Commissions par catégorie
  • Reversements effectués aux vendeurs
  • Remboursements et leurs raisons
  • Projection de croissance
  • Export CSV/Excel pour comptabilité

👥 Gestion des admins

  • Créer un compte admin
  • Définir les permissions par admin
  • Suspendre / Réactiver un admin
  • Voir l'activité de chaque admin
  • Supprimer (double confirmation)
  • Historique des connexions

📋 Audit & Sécurité

  • Log de toutes les actions admin
  • IP de connexion tracée
  • Détection connexions suspectes
  • Export du journal d'audit
  • Alertes en cas d'activité anormale
  • Sauvegarde des données automatique
